# Systemd-based Local Development

This directory contains systemd user unit files for running Flow components locally.

## Overview

Instead of using Tilt or tmux scripts, we use:
- **systemd user units** - manage service lifecycle, dependencies, restarts, and logging
- **mise tasks** - setup environment, generate configs, start/stop services
- **Debug builds on-demand** - services use `cargo run` / `go run` for auto-rebuild

## Architecture

```
mise tasks
  ├─ Generate environment files (~/flow-local/env/*.env)
  ├─ Generate wrapper script (~/flow-local/bin/run-in-repo.sh)
  └─ Link systemd units and start services

systemd units (this directory)
  ├─ flow-etcd.service              # Single etcd instance
  └─ flow-gazette@.service          # Template for gazette brokers

Wrapper script
  └─ ~/flow-local/bin/run-in-repo.sh
     ├─ Sources common.env (PATH, REPO_ROOT, TLS paths)
     ├─ Changes to REPO_ROOT
     └─ Execs the service command
```

## Getting Started

### 1. Start Etcd

```bash
mise run dev:etcd
```

This will:
- Generate common environment file at `~/flow-local/env/common.env`
- Generate TLS certificates if needed
- Generate etcd-specific environment
- Link the systemd unit
- Start etcd

### 2. Start Gazette Brokers

Start individual gazette brokers by specifying cluster name and port:

```bash
# Start local-cluster brokers
mise run dev:gazette local-cluster 8080
mise run dev:gazette local-cluster 8081
mise run dev:gazette local-cluster 8082
mise run dev:gazette local-cluster 8083
```

Or start a different cluster:

```bash
# Start other-cluster brokers
mise run dev:gazette other-cluster 8180
mise run dev:gazette other-cluster 8181
```

Each broker runs independently. The cluster name determines the etcd prefix (`/gazette/<cluster>`).

### 3. View Logs

```bash
# All Flow services
mise run dev:logs

# Specific service
mise run dev:logs flow-etcd

# Specific gazette instance
mise run dev:logs flow-gazette@local-cluster-8080
```

Or use journalctl directly:
```bash
journalctl --user -u flow-etcd -f
journalctl --user -u 'flow-gazette@*' -f
```

### 4. Check Status

```bash
systemctl --user status flow-etcd
systemctl --user status flow-gazette@local-cluster-8080
systemctl --user list-units 'flow-*'
```

### 5. Stop Services

```bash
# Stop all Flow services
mise run dev:stop

# Stop specific service
systemctl --user stop flow-etcd

# Stop specific gazette instance
systemctl --user stop flow-gazette@local-cluster-8080

# Stop all gazette instances for a cluster
systemctl --user stop 'flow-gazette@local-cluster-*'
```

## Environment Files

Environment is split into hierarchical files:

### Common Environment (`~/flow-local/env/common.env`)

Generated by `dev:env-common`, contains:
- `REPO_ROOT` - absolute path to flow repository
- `PATH` - includes mise-managed tools
- `CARGO_TARGET_DIR`, `ROCKSDB_LIB_DIR`, etc.
- TLS certificate paths

### Service-Specific Environment

Each service has its own env file with instance-specific configuration:
- `~/flow-local/env/etcd.env` - etcd listen addresses
- `~/flow-local/env/gazette-<cluster>-<port>.env` - broker config, auth keys, etcd prefix

## Unit File Details

### flow-etcd.service

Single etcd instance on ports 2379 (client) and 2380 (peer).

- Data directory: `~/flow-local/etcd`
- Environment: `~/flow-local/env/etcd.env`

### flow-gazette@.service (Template)

Template unit for gazette brokers. Instance name format: `<cluster>-<port>`

Example instances:
- `flow-gazette@local-cluster-8080.service`
- `flow-gazette@other-cluster-8180.service`

Each instance:
- Depends on `flow-etcd.service`
- Loads environment from `~/flow-local/env/gazette-<cluster>-<port>.env`
- Runs `go run ./go/gazette serve` (rebuilds on start if code changed)
- Auto-restarts on failure

## Wrapper Script

`~/flow-local/bin/run-in-repo.sh` is a simple wrapper that:
1. Sources `~/flow-local/env/common.env`
2. Changes to `$REPO_ROOT`
3. Executes the command

This allows systemd units to run commands in the correct directory with the mise environment, without needing `WorkingDirectory` or `EnvironmentFile` directives in every unit.

## Advantages Over Tilt

- **Native systemd features**: dependencies, health checks, auto-restart, rate limiting
- **Better log management**: journalctl with filtering, timestamps, persistent logs
- **Service status**: `systemctl status` shows detailed state
- **Lightweight**: no additional daemon (Tilt) running
- **Debug builds**: `cargo run` / `go run` rebuild on changes
- **Flexible**: easy to add/remove instances or change configuration

## Adding New Services

To add a new service (e.g., reactor):

1. **Create systemd unit**: `systemd/flow-reactor@.service`
   ```systemd
   [Unit]
   Description=Flow Reactor (%i)
   After=flow-etcd.service

   [Service]
   Type=simple
   EnvironmentFile=%h/flow-local/env/reactor-%i.env
   ExecStart=%h/flow-local/bin/run-in-repo.sh go run ./go/flowctl-go serve consumer
   Restart=on-failure
   ```

2. **Create mise task**: `mise/tasks/dev/reactor`
   ```bash
   #!/usr/bin/env bash
   #MISE description="Start reactor via systemd"
   #MISE depends=["dev:etcd"]

   CLUSTER="${1:-}"
   PORT="${2:-}"

   # Generate env file at ~/flow-local/env/reactor-${CLUSTER}-${PORT}.env
   # Link unit file
   # Start service
   ```

3. **Start it**: `mise run dev:reactor local-cluster 9000`

## Troubleshooting

### Service won't start

Check logs:
```bash
journalctl --user -u flow-etcd -n 50
systemctl --user status flow-etcd
```

### Environment issues

Inspect environment files:
```bash
cat ~/flow-local/env/common.env
cat ~/flow-local/env/gazette-local-cluster-8080.env
```

### Wrapper script issues

Test the wrapper directly:
```bash
~/flow-local/bin/run-in-repo.sh env | grep REPO_ROOT
~/flow-local/bin/run-in-repo.sh pwd
```

### Reset everything

```bash
mise run dev:stop
rm -rf ~/flow-local
rm -rf ~/.config/systemd/user/flow-*
systemctl --user daemon-reload
```
