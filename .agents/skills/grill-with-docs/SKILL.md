# Grill with Docs

Run a `/grilling` session, reading `docs/platform/` through the
[platform docs doctrine](../platform-docs/DOCTRINE.md) as your lens.
The glossary is your consistency oracle: challenge every term against it, and
stress-test each decision for what it would change in the porcelain.

You are **not** writing docs. Instead, accrue the **docs delta** — the running
brief of intended changes to `docs/platform/`, held in the conversation. When a
decision would add, rename, or alter a concept, term, node, or relationship,
record the intended change in doctrine terms — which node, the nature of the
change, and the decision that forces it — and restate the growing delta so it
stays salient.

The grill is done when we reach shared understanding **and** every decision's
porcelain implication is accounted for in the delta — accrued or explicitly
dismissed.
